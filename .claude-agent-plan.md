## Goal

Let anyone on the team open a GitHub issue (bug or feature) for the Buena Copa Shopify theme, `@claude`-mention to trigger the agent, get an isolated Shopify preview theme per PR for review, and ship to prod by merging to `main` (which already auto-deploys via Shopify's GitHub integration).

## Guardrails for CLAUDE.md

- Never edit `config/settings_data.json` (managed by Shopify theme customizer — per `CONTRIBUTING.md`)
- Never edit `assets/tailwind.min.css` directly (compiled output)
- Never run `shopify theme push --live` — prod deploys only via `main` merge
- Mobile-first: 70%+ of Buena Copa traffic is mobile (per `CONTRIBUTING.md`)
- Theme stack: Shopify Online Store 2.0, Tailwind (compiled), vanilla JS, Liquid
- Spanish is the default locale (`locales/es.default.json`); English is secondary (`locales/en.json`)
- Commit message format: `fix:`, `feat:`, `perf:`, `docs:` prefix (per `CONTRIBUTING.md`)

```

```

The preview workflow should capture the theme ID from `shopify theme push --unpublished --json` on first push, persist it (PR comment marker, or a file keyed by PR number in a cache), and reuse it on subsequent pushes so the URL stays stable across commits.
